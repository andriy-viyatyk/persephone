# 3. `app.recent` — IRecentFiles

**Phase:** 1c (Independent — depends only on filesModel for persistence, not on other interfaces)

**Status:** Complete

**API Reference:** [../api-reference/recent.md](../api-reference/recent.md)

---

## Summary

Moved all recent files logic from `/src/renderer/store/recent-files.ts` into `/src/renderer/api/recent.ts`. The implementation class IS the interface — no thin wrapper. Updated all 4 consumers, deleted the old store.

---

## What Was Done

### Code Moved to `/api/recent.ts`

| Logic | Notes |
|---|---|
| `RecentFilesState` type + `defaultRecentFilesState` | State shape |
| `recentFileName` constant | File name for persistence (`recentFiles.txt`) |
| `TGlobalState` for reactive state | Replaces `TModel` base class — state declared directly |
| `files` getter | Reads from state |
| `useFiles()` | React hook for reactive reading (not in `.d.ts`) |
| `load()` — read from disk, parse lines | File → state |
| `add()` — dedup, 100-cap, persist | Core logic |
| `remove()` — filter, persist | Core logic |
| `clear()` — empty state, persist | Core logic |

### Consumer Updates (4 files)

All changed from `import { recentFiles } from "../store/recent-files"` (or barrel) to `import { recent } from "../api/recent"`.

| # | File | Usage |
|---|---|---|
| 1 | `editors/text/TextPageModel.ts` | `recent.add()` on save, `recent.remove()` + `recent.add()` on rename |
| 2 | `features/sidebar/RecentFileList.tsx` | `recent.load()`, `recent.useFiles()`, `recent.remove()` |
| 3 | `features/sidebar/MenuBar.tsx` | `recent.clear()` |
| 4 | `store/pages-store.ts` | `recent.add()` when opening files |

### Files Deleted

| File | Reason |
|---|---|
| `/src/renderer/store/recent-files.ts` | All logic moved to `/api/recent.ts` |

### Barrel Export Updates

| File | Change |
|---|---|
| `/src/renderer/store/index.ts` | Removed `recentFiles` re-export |

---

## Key Decisions

1. **No `TModel` base class** — `TGlobalState` used directly as a private field
2. **`useFiles()` method** replaces `recentFiles.state.use((s) => s.files)` — cleaner API, hides internal state structure
3. **`.useFiles()` not in `.d.ts`** — hidden from scripts per React hooks safety decision
4. **`filesModel` dependency accepted** — temporary, will update when Phase 2 is revised
