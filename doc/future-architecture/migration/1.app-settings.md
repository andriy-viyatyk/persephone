## Implementation Plan

### 1.1. Define ISettings interface

**File:** `/src/renderer/api/types.ts` (add to existing)

```typescript
export interface ISettings {
  /** Current theme name */
  readonly theme: string;

  /** Get a setting value by key */
  get<T = any>(key: string): T;

  /** Set a setting value. Persisted automatically. */
  set<T = any>(key: string, value: T): void;

  /** Fires when any setting changes */
  readonly onChanged: IEvent<{ key: string; value: any }>;
}
```

### 1.2. Implement Settings wrapper

**File:** `/src/renderer/api/settings.ts`

```typescript
import { appSettings } from "../store/app-settings";
import type { ISettings } from "./types";
import { wrapSubscription } from "./types";

class Settings implements ISettings {
  get theme(): string {
    return appSettings.get("theme");
  }

  get<T = any>(key: string): T {
    return appSettings.get(key as any);
  }

  set<T = any>(key: string, value: T): void {
    appSettings.set(key as any, value as any);
  }

  readonly onChanged: IEvent<{ key: string; value: any }>;

  constructor() {
    // Note: appSettings currently doesn't have a global onChanged event.
    // This will need to be added — either:
    //   a) Add a Subscription to AppSettings that fires on every set()
    //   b) Create a new Subscription here that intercepts set() calls
    // Decision: Add to AppSettings directly (cleaner).
    this.onChanged = wrapSubscription(appSettings.onChanged);
  }
}

export const settings = new Settings();
```

**Required change to `app-settings.ts`:** Add `onChanged` Subscription that fires whenever `set()` is called.

### 1.3. Add `onChanged` to AppSettings

**File:** `/src/renderer/store/app-settings.ts`

Add a `Subscription` to `AppSettings` class:

```typescript
// In AppSettings class:
readonly onChanged = new Subscription<{ key: string; value: any }>();

// In set() method, after the value is stored:
this.onChanged.fire({ key, value });
```

This is a small, backward-compatible addition.

### 1.4. Register on `app` root

**File:** `/src/renderer/api/app.ts`

```typescript
import { settings } from "./settings";

// Update IApp interface (in types.ts):
export interface IApp {
  readonly version: string;
  readonly settings: ISettings;
}

// In App class:
get settings() { return settings; }
```

### 1.5. Verify

1. Run the app
2. In script panel: `app.settings.theme` → returns current theme
3. `app.settings.get("theme")` → same
4. `app.settings.set("theme", "monokai")` → theme changes
5. `app.settings.onChanged.subscribe(e => console.log(e))` → fires on changes
6. Existing `appSettings.get()` / `appSettings.use()` still work everywhere
