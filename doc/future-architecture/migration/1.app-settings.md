### Method Overloads

`get()` and `set()` use TypeScript method overloads for type safety:

```typescript
get<K extends AppSettingsKey>(key: K): AppSettingsState["settings"][K];
get<T = any>(key: string): T;

set<K extends AppSettingsKey>(key: K, value: AppSettingsState["settings"][K]): void;
set<T = any>(key: string, value: T): void;
```

**Important:** The specific `AppSettingsKey` overload must be declared **first** ‚Äî TypeScript resolves overloads in declaration order.

## Key Decisions

1. **No `TModel` base class** ‚Äî `TGlobalState` used directly as a private field
2. **Overload order matters** ‚Äî specific `AppSettingsKey` overload before generic `string` overload
3. **`BrowserProfile` and `AppSettingsKey` types** moved to `/api/settings.ts` ‚Äî they're settings-related
4. **`.use()` not in `.d.ts`** ‚Äî hidden from scripts per React hooks safety decision
5. **`filesModel` dependency accepted** ‚Äî temporary, will update when Phase 2 is revised
