# 5. `app.window` — IWindow

**Phase:** 2b (Window)

**Status:** Complete (revised in US-047)

**API Reference:** [../api-reference/window.md](../api-reference/window.md)

---

## What

Window management API wrapping IPC window endpoints:
- Window actions: minimize, maximize, restore, close
- Zoom: zoom in/out, reset, current level
- State: isMaximized (reactive)
- Multi-window: openNew
- Window identity: windowIndex

Phase 2 subset only. Methods that require `app.pages` (like `showWindowPage`, page-moving between windows) are deferred to Phase 4.

---

## Implementation

### Files

| File | Purpose |
|------|---------|
| `/src/renderer/api/types/window.d.ts` | IWindow interface (added `windowIndex`) |
| `/src/renderer/api/window.ts` | Window implementation (added `windowIndex` via IPC, renamed export to `appWindow`) |

### Delegation

- `minimize()` / `maximize()` / `restore()` / `close()` → `api.minimizeWindow()` / etc.
- `zoom(delta)` → `api.zoom(delta)`
- `resetZoom()` → `api.resetZoom()`
- `openNew(filePath?)` → `api.openNewWindow(filePath)`
- `isMaximized` → cached from `rendererEvents.eWindowMaximized` subscription
- `zoomLevel` → cached from `rendererEvents.eZoomChanged` subscription
- `windowIndex` → fetched once via `api.getWindowIndex()` during async init

### Reactive State

The `Window` class subscribes to IPC events in its constructor:
- `eWindowMaximized` → updates `_isMaximized`
- `eZoomChanged` → updates `_zoomLevel`
- `_initWindowIndex()` → async, fetches `windowIndex` once via IPC

### Consumer Updates (US-047 Phase B)

4 files migrated from `fs.windowIndex` to `appWindow.windowIndex`:
- `app/EventHandler.tsx` — drag-drop cross-window detection
- `app/MainPage.tsx` — content drop grouping
- `features/tabs/PageTab.tsx` — tab drag source/target identification
- `features/sidebar/OpenTabsList.tsx` — current window identification

### Export Naming

The singleton is exported as `appWindow` (not `window`) to avoid shadowing the global `window` object. `app.ts` imports it as `{ appWindow }` and assigns to `this._window`.

---

## Design Decisions

1. **Phase 2 subset** — only window actions, zoom, openNew, and windowIndex. Page-moving between windows deferred to Phase 4 (requires `app.pages`).
2. **Reactive properties** — `isMaximized` and `zoomLevel` cached from event subscriptions, returned synchronously.
3. **Fire-and-forget actions** — window actions (`minimize`, `maximize`, etc.) don't await IPC responses.
4. **No event cleanup** — subscriptions live for the lifetime of the window. Since each window has its own `app` instance, cleanup happens when the window closes.
5. **windowIndex on IWindow** — semantically correct (window identity, not file system). `FileSystem` keeps its own private `_windowIndex` for path resolution.
6. **Export as `appWindow`** — avoids shadowing global `window` which is used extensively (e.g., `window.addEventListener`, `window.electron`).

---

## Deferred to Phase 4

The following will be added when `app.pages` is implemented:
- `showWindowPage(windowIndex, pageId)` — navigate to a specific page in another window
- Page-moving between windows (requires page identity via `app.pages`)
