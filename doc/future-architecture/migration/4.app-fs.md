## What

Unified file system API combining:
- Direct Node.js file operations (read/write/exists/delete) via `nodeUtils`
- IPC-based dialogs (open/save/folder) via `api.*`
- OS integration (commonFolder, showInExplorer, showFolder) via `api.*`
- Path resolution (resolveDataPath, resolveCachePath) via `filesModel.state`

The API serves both scripts and application code. Two read methods: `read()` returns a simple string (for scripts), `readFile()` returns `{ content, encoding }` (for app code needing encoding info, e.g., TextPageModel).

### Delegation

- `read()` / `readFile()` â†’ `nodeUtils.loadStringFile()` (BOM detection, jschardet, iconv-lite)
- `readBinary()` â†’ `fs.readFileSync()`
- `write()` â†’ `nodeUtils.preparePath()` + `nodeUtils.saveStringFile()`

- `exists()` â†’ `nodeUtils.fileExists()`
- `delete()` â†’ `nodeUtils.deleteFile()`
- `resolveDataPath()` / `resolveCachePath()` â†’ reads from `filesModel.state`
- `commonFolder()` â†’ `api.getCommonFolder()` (IPC)
- `showOpenDialog()` / `showSaveDialog()` / `showFolderDialog()` â†’ `api.showOpenFileDialog()` / `api.showSaveFileDialog()` / `api.showOpenFolderDialog()` (IPC). Returns `null` instead of `undefined` on cancel.
- `showInExplorer()` / `showFolder()` â†’ `api.showItemInFolder()` / `api.showFolder()` (IPC)

## Design Decisions

1. **Two read methods** â€” `read()` returns `string` (simple, for scripts), `readFile()` returns `ITextFile` (for app code needing encoding)
2. **Wraps nodeUtils, not reimplements** â€” delegates to existing encoding-aware read/write. End-state cleanup will absorb nodeUtils into fs.ts (see migration README).
3. **Path resolution from filesModel.state** â€” reads cached `dataPath`/`cachePath`/`windowIndex`. filesModel is initialized during main bundle load, before `initServices()` runs.
4. **Dialog return types** â€” `null` on cancel (not `undefined`), consistent with the API convention.
5. **write/writeBinary auto-create directories** â€” calls `nodeUtils.preparePath()` before writing.

## Consumer Migration (Gradual, Later)

Not required now. Existing `nodeUtils.*`, `filesModel.*`, and `api.*` continue to work.

Future migration examples:
```typescript

// After
const { content, encoding } = await app.fs.readFile(path);
```

