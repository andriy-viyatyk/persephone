# 7. `app.shell` — IShell

**Phase:** 3b (Shell)

**Status:** Complete

**API Reference:** [../api-reference/shell.md](../api-reference/shell.md)

---

## What

OS integration and utility services:
- Open URLs in the default browser (`openExternal`)
- Content encryption/decryption (AES-GCM via Web Crypto API)
- Runtime version info and update checking

Organized as `api/shell/` subfolder — each concern in its own module, composed by `index.ts`.

### Architecture

```
api/shell/
├── encryption.ts      # EncryptionService implements IEncryptionService
│   ├── encrypt(text, password) → AES-GCM encrypt
│   ├── decrypt(text, password) → AES-GCM decrypt (version dispatch)
│   ├── isEncrypted(text) → checks ENC-vNNN: prefix
│   └── (internal: getKeyFromPassword, arrayBufferToBase64, encryptionVersion)
│
├── version.ts         # VersionService implements IVersionService
│   ├── runtimeVersions() → api.getRuntimeVersions()
│   └── checkForUpdates(force?) → api.checkForUpdates() → maps to IUpdateInfo
│
├── shell-calls.ts     # Direct OS calls
│   └── openExternal(url) → require("electron").shell.openExternal(url)
│
└── index.ts           # Shell class implements IShell, composes sub-modules
    ├── readonly encryption = EncryptionService singleton
    ├── readonly version = VersionService singleton
    └── openExternal(url) → delegates to shell-calls
```

### IUpdateInfo Mapping

`VersionService.checkForUpdates()` maps the IPC `UpdateCheckResult` (with nested `releaseInfo`) to a flat `IUpdateInfo`:

| IUpdateInfo field | Source |
|-------------------|--------|
| `currentVersion` | `result.currentVersion` |
| `latestVersion` | `result.latestVersion` |
| `updateAvailable` | `result.updateAvailable` |
| `releaseUrl` | `result.releaseInfo?.htmlUrl` |
| `releaseVersion` | `result.releaseInfo?.version` |
| `publishedAt` | `result.releaseInfo?.publishedAt` |
| `releaseNotes` | `result.releaseInfo?.body` |
| `error` | `result.error` |
